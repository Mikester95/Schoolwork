Computer Science Lessons

Don't know where to start? Don't worry! When you try to access a lesson, you will be informed by its prerequisites. If you're new, I recommend you start with Miscellaneous : Data Representations .

  • Data Structures

    Arguably the most important section here, data structures are all about laying out information in the computer's memory so that your program can access the data in the fastest and/or most space efficient manner for the given task.

    • Basic Data Structures
    • Implementing Unordered Sets
    • Implementing Ordered Sets
    • Segment Structures

  • Numerical Algorithms

    This section deals with the more mathematical aspects of computer science. If you aren't good at maths, you're about to become. Not only is it easier to visualize math problems through the eyes of a computer scientist, but you'll also discover that you need a computer to do the heavy computation in order to put all that nice mathematical knowledge to good use.

    • Arithmetic
    • Number Factorization
    • Combinatorics
    • Computational Algebra

  • Graph Algorithms

    Graphs are the computer's way to illustrate sets and connections/relations between elements of these sets. Starting from this simple idea, there are a lot of interesting algorithms that arise when you model certain problems with graphs.

    • Traversals
    • Shortest Paths
    • Minimum Spanning Trees
    • Connectivity
    • Maximum Flow and Maximum matching
    • Algorithms on Trees

  • String Algorithms

    Strings are simply sequences of symbols that computers work with to represent information. When talking about sequences of something, things like patterns and reccurences inevitably come into question. This section is all about algorithms that can detect these things.

    • Pattern Matching
    • Data structures for pattern matching

  • Geometry Algorithms

    Probably the oldest branch of mathematics, geometry deserves to be differentiated from its sister branches, simply because it is drastically enhanced by computer science and is one of today's most important areas of algorithmics.

    • Representing and operating on sets of points (segments, lines, planes)
    • Using vectors
    • Convex Hull Algorithms
    • Polygon Algorithms
    • Voronoi Diagrams

  • Algorithmic Techniques

    If you go through the sections above, you'll notice that not every algorithm is a new, special idea. Patterns arise when trying to conceive solutions. These patterns can be mastered, and this is exactly what this section is about.

    • Brute Force
    • Backtracking
    • Greedy
    • Dynamic Programming

  • Miscellaneous

    This section will contain anything that does not fit in any other section and isn't vast enough to have its own. Don't be fooled! You'll find some pretty cool things here. One of them is algorithm analysis, i.e. how to evaluate the speed and space usage of your programs.

    • Data Representations (basic computer data types and operations)
    • Algorithm Analysis
    • Binary Search
    • Sorting Algorithms
    • Game Theory
